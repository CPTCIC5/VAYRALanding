DESIGN DIRECTION
================================================================

AESTHETIC: Military-grade dark minimalism with Indian soul.
Think CX2 meets Anduril meets something that could only come
from India. Not corporate. Not startup-y. Serious. Purposeful.

COLOR PALETTE:
--bg:          #000000  (pure black)
--surface:     #0A0A0A  (cards, sections)
--border:      #1A1A1A  (subtle borders)
--text:        #F0EDE8  (warm off-white, not pure white)
--text-muted:  #6B6B6B  (secondary text)
--accent:      #C8A96E  (deep gold/amber — Indian warmth)
--accent-dim:  #7A6540  (dimmed gold for borders/glows)
--danger:      #E84040  (threat indicator red)
--safe:        #40A870  (classified green)

TYPOGRAPHY:
- Display/Hero: "Bebas Neue" or "Anton" — bold, military, commanding
- Headings: "Space Mono" — technical, precise, monospace character
- Body: "DM Sans" — clean, readable, modern
- Accent labels: "JetBrains Mono" — code/data feel for stats and numbers

CRITICAL PERFORMANCE RULES:
- Every animation must use transform and opacity only (GPU composited)
- No layout-triggering animations (no width/height/top/left)
- Use will-change: transform sparingly and only on active animated elements
- Three.js scene must use instanced meshes, not individual objects
- Lazy load everything below the fold
- Target 60fps on mid-range devices
- No animation should block interactivity

================================================================
ANIMATION MASTER PLAN
================================================================

PERFORMANCE RULES (non-negotiable):
- All Framer Motion animations: use transform + opacity only
- GSAP animations: force3D: true on all tweens
- Three.js: requestAnimationFrame, dispose unused geometries
- Images: next/image with lazy loading and blur placeholder
- Fonts: next/font with display: swap
- No animation starts until element is in viewport
- Use IntersectionObserver for all scroll triggers

ANIMATION INVENTORY:

1. PAGE LOAD (runs once, <1200ms total):
   - Nav fades in: 0ms delay, 400ms duration
   - Hero label slides up: 200ms delay
   - Hero headline lines reveal (clip-path): 400/600/800ms stagger
   - Hero subline fades up: 1000ms delay
   - Hero CTA fades up: 1200ms delay
   - Three.js scene fades in: 0ms, 2000ms duration (slow, atmospheric)

2. SCROLL ANIMATIONS (Framer Motion whileInView):
   - Mission strip: word by word reveal, 50ms stagger
   - Problem section paragraphs: fade + translateY(20px), 200ms stagger
   - Stats: counter animation, 1000ms ease-out
   - Product cards: translateY(40px) → 0, 150ms stagger
   - Case columns: clip-path bottom reveal, 200ms stagger
   - Vision typewriter: triggered by scroll, 50ms per character
   - Team cards: translateX (left/right), simultaneous

3. HOVER STATES (CSS transitions, 200ms ease):
   - Nav links: subtle gold underline slides in from left
   - Product cards: translateY(-4px), gold glow
   - Buttons: background fill slides up (clip-path trick)
   - Footer links: opacity 0.5 → 1

4. THREE.JS SCENE (hero background):
   - Wireframe radar/globe, rotates at 0.0005 rad/frame (very slow)
   - Points geometry for stars/particles (500 points max)
   - Fog to fade edges
   - Pauses when tab is not visible (document.visibilityState)
   - Destroys and recreates on window resize (debounced 500ms)

/* Scanline effect — apply to hero only */
.scanlines::after {
  content: '';
  position: absolute;
  inset: 0;
  background: repeating-linear-gradient(
    0deg,
    transparent,
    transparent 2px,
    rgba(0, 0, 0, 0.03) 2px,
    rgba(0, 0, 0, 0.03) 4px
  );
  pointer-events: none;
  z-index: 1;
}

/* Grain texture overlay */
.grain::before {
  content: '';
  position: fixed;
  inset: -50%;
  width: 200%;
  height: 200%;
  background-image: url("data:image/svg+xml,..."); /* SVG noise */
  opacity: 0.025;
  animation: grain 0.5s steps(1) infinite;
  pointer-events: none;
  z-index: 9999;
}

@keyframes grain {
  0%, 100% { transform: translate(0, 0); }
  10% { transform: translate(-5%, -10%); }
  30% { transform: translate(3%, -15%); }
  50% { transform: translate(12%, 9%); }
  70% { transform: translate(9%, 4%); }
  90% { transform: translate(-1%, 7%); }
}

================================================================
SPECIFIC INSTRUCTIONS FOR CLAUDE CODE
================================================================

1. Start with globals.css and layout.tsx — get the fonts and
   CSS variables in place before building any components.

2. Build Nav first. Make sure scroll opacity works before moving on.

3. Build Hero second. Get the Three.js scene working as a
   dynamically imported client component (use next/dynamic with
   ssr: false). Test that it doesn't cause hydration errors.

4. Build all remaining sections in order top to bottom.

5. Add Framer Motion animations last, after all content is
   confirmed correct. Don't animate broken content.

6. Performance check before finishing:
   - Open Chrome DevTools → Performance tab
   - Record scroll through full page
   - Confirm no frames drop below 55fps
   - If they do, reduce Three.js particle count first,
     then simplify GSAP ScrollTrigger instances

7. Mobile check:
   - Nav collapses to hamburger at <768px
   - Hero text sizes down gracefully (use clamp())
   - Product cards become single column horizontal scroll
   - Three.js scene renders at 0.5x resolution on mobile
     (use devicePixelRatio capped at 1 on mobile)

8. Final check — make sure these work:
   - Custom cursor (desktop only, hidden on touch devices)
   - Grain animation is subtle (barely perceptible)
   - Scanline is subtle (barely perceptible)
   - All animations respect prefers-reduced-motion
   - No horizontal scroll on any viewport
